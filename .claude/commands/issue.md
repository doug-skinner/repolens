Create a GitHub issue in this repo. Keep the title and body as short and concise as possible — no templates, no checklists, no verbose descriptions.

Use `gh issue create` with the appropriate label(s) from the repo's standard set:
- `bug` — Something isn't working
- `fix` — Bug fix
- `feature` — New feature
- `enhancement` — Improvement to existing feature
- `chore` — Maintenance / tooling
- `documentation` — Improvements or additions to documentation
- `question` — Further information is requested
- `help wanted` — Extra attention is needed
- `good first issue` — Good for newcomers

Rules:
1. Title: one short sentence, no period.
2. Body: 1-3 sentences max. Just enough context to act on. No headers, no markdown formatting, no acceptance criteria.
3. Pick the single most appropriate label (use multiple only when clearly warranted).
4. Always confirm the title, body, and label(s) with the user before creating.
